# 바이럴 게시글 작성 가이드라인

## 플랫폼: 펨코 (fmkorea.com)

### 톤 & 매너
- **~임 / ~음 / ~함** 체 (반말 서술형)
- 개발 뉴비 컨셉 유지 ("바이브 코딩", "AI한테 시킴")
- 자학적 유머 섞기 (질 걸 알지만, 아무도 안 쓸 수도 있지만)
- 과하게 겸손 + 은근 자랑 밸런스

### 구조
1. **도입** — 왜 이 글을 올리는지 (1~2줄)
2. **업데이트 요약** — 번호 매겨서 간결하게
3. **스크린샷** — `![설명]` 형태로 위치 표시 (복붙 시 이미지 삽입 위치 마커)
4. **로드맵/예고** — 다음에 뭘 할 건지 가볍게
5. **앱 링크 + 이전 글 링크**

### 복붙 후처리 체크리스트
- [ ] `![Screenshot: ...]` 위치에 실제 스크린샷 삽입
- [ ] `[앱 링크]`, `[이전 글 링크]` 실제 URL 교체
- [ ] 제목 별도 입력 (본문에 제목 포함하지 않음)
- [ ] 말머리/태그 설정

### 파일 네이밍
- `v{버전}.md` — 예: `v1.0.md`, `v1.1.md`
- 버전은 메이저 기능 단위가 아닌 **게시글 시리즈 번호**로 관리

### 참고
- 마크다운 서식은 펨코에서 안 먹으므로 **순수 텍스트 + 번호 리스트** 위주
- 굵은 글씨(`**`)도 펨코에서 안 되므로 사용하지 않음
- 이미지는 별도 업로드 → 본문 삽입 방식
